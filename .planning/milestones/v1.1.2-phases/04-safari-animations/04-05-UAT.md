## How to Run the Safari UAT

### Prerequisites

1. macOS with Safari latest installed.
2. A built site running locally. **Recommended — production-parity build** (avoids hydration-flash false positives per RESEARCH.md §A4):

   ```bash
   pnpm build && pnpm start
   ```

   Alternative (dev server, faster iteration but less production-accurate):

   ```bash
   pnpm dev
   ```

3. Open Chrome side by side with Safari for item 4 comparison.

### Procedure — complete ALL four items

For each item below: perform the check, mark PASS or FAIL in the table, add notes if anything is notable (even on PASS).

#### Item 1 — Reveal fades have no flicker (SAFARI-01, ANI-02)

- Hard-reload Safari (Cmd+Option+R) at http://localhost:3000.
- Scroll slowly from top to bottom.
- Observe each migrated section as it enters the viewport:
  - Team section photo.
  - Philosophy team card (card 03 "Команда").
  - Cases section — vertical cards as they scroll into view.
  - Cases section — ad cards as they scroll into view.
- PASS if: each card fades in over ~220ms with a smooth opacity ramp, no mid-animation flash, no opacity jump, no flicker of the underlying black `bg-[#0d0300]` after the image loads.
- FAIL if: any of the above flickers, jumps, or replays on subsequent scroll.
- Repeat at viewport widths 1440, 1024, 768, 480, 360 (resize Safari window or use Responsive Design Mode).

#### Item 2 — ANI-01 pin holds position (SAFARI-02, ANI-01)

- At viewport width ≥1024 in Safari, scroll into the cases section.
- The vertical slider, divider, and "Рекламные кейсы" header should pin and stay fixed at the top of the viewport while the collapse animation progresses, then release and continue with the ad strip expanded.
- PASS if: pinned stack stays at the top throughout the pin zone; no upward jumps or unpin glitches; weights transition smoothly as scroll progresses; scroll input maps 1:1 to progress.
- FAIL if: pinned stack jumps up or down at pin entry/exit; content "falls off"; progress stutters or overshoots; scroll feels unresponsive inside the pin zone.

#### Item 3 — Hover transitions without flash (ANI-02)

- Hover over cases cards (vertical + ad) in Safari.
- Watch the grayscale → color transition and the `mix-blend-color` overlay fade.
- PASS if: grayscale-to-color transition is smooth; overlay opacity changes cleanly without flickering; no "pop" frame at the start of the hover.
- FAIL if: a color or saturation flash precedes the transition; the transition stutters.

#### Item 4 — Scroll playback parity vs Chrome (SAFARI-01, SAFARI-02)

- Open the same page in Chrome side by side with Safari.
- Scroll through cases pin zone, philosophy stack, and all reveal sections at similar speed in each browser.
- PASS if: Safari behavior is indistinguishable from Chrome to a careful human observer; no visual glitches that only Safari exhibits.
- FAIL if: Safari clearly diverges from Chrome in a way a client would notice.
