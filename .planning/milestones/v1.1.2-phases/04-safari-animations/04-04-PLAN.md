<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server render → client hydration | `motion.article` with `initial opacity 0` renders inline in SSR HTML, then client hydrates and animates to 1. |
| CSS hover layer ↔ motion animation | Both systems animate different properties (hover: filter/opacity on children; motion: opacity on root) — orthogonal, no fight expected. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08 | Tampering (visual regression) | Removing `imageLoaded` state might cause a pre-image flash on very slow networks | accept | D-05 explicitly allows simplification. Assumption A1 in RESEARCH.md — if Safari UAT (Plan 04-05) shows FOUC, add `placeholder="blur"` as a follow-up. Not mitigated here to keep the diff minimal. |
| T-04-09 | Denial-of-service | IntersectionObserver per card (potentially 10+ cards on a page) | mitigate | `viewport.once: true` — observers disconnect after first trigger. Cost is bounded. motion uses a shared root observer when possible (per motion.dev docs). |
| T-04-10 | Elevation of privilege | `onClick` / `onKeyDown` handlers on `motion.article` | accept | Same handlers that were on `<article>` before — no new attack surface. `role="button"` and `tabIndex` preserved. |
</threat_model>
