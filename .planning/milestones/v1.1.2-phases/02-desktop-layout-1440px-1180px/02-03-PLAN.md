**Шаг 1 — Диагностика (читать перед изменением):**
Проверить строку 118: `<div className="relative mx-auto w-[1024px] min-w-[1024px]">` — это контейнер для карточек-стека. Маркиза находится ВНУТРИ этого контейнера (строки ~369-382).

**Шаг 2 — Применить 100vw breakout на wrapper маркизы (предпочтительный подход per D-09):**

Найти wrapper маркизы (~строка 379):
```tsx
<div className="relative mt-5 h-[400px] w-full max-w-full overflow-x-clip">
  <PhilosophyClientsMarquee1024 />
</div>
```

Заменить на 100vw breakout:
```tsx
<div
  className="relative mt-5 h-[400px]"
  style={{
    width: "100vw",
    marginLeft: "calc(50% - 50vw)",
  }}
>
  <PhilosophyClientsMarquee1024 />
</div>
```

Это убирает `overflow-x-clip` и `max-w-full`, применяет breakout через negative margin + 100vw.

**Шаг 3 — Проверить на горизонтальный скроллбар:**
Если после этого изменения появляется горизонтальный скроллбар на странице — значит, нет `overflow-x: hidden` на ancestor. Тогда применить запасной вариант:

ЗАПАСНОЙ ВАРИАНТ: Найти элемент с `overflow-x-clip` или `overflow-hidden` на строке ~371 (section wrapper):
```tsx
<div className="relative z-[5] w-full overflow-x-visible overflow-y-visible rounded-bl-[60px] rounded-br-[60px] bg-white pb-10">
```
Эта строка уже имеет `overflow-x-visible` — значит, ограничитель выше. Искать в `<section>` (строка 114): `overflow-visible` тоже есть. Проверить `<div className="w-full rounded-bl-[60px] rounded-br-[60px] bg-white">` (строка 117) — если там нет overflow-hidden, значит скроллбар будет виден только если body/html не имеет `overflow-x: hidden`.

Если скроллбар появился: добавить `overflow-x: hidden` на `<section>` (строка 114) или на wrapper `<div className="w-full min-w-0">` (строка 112). НЕ добавлять на прямой родитель маркизы.
