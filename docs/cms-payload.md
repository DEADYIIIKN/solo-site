# Контент: Payload CMS 3

## Что сделано

- **Payload CMS 3** встроен в этот Next.js-проект (без отдельного бэкенда).
- Админка: **`/admin`** (после первого запуска создайте пользователя).
- БД по умолчанию: **SQLite** (`payload.db` в корне проекта, в `.gitignore`).
- Коллекции:
  - **users** — вход в админку
  - **media** — загрузка картинок
  - **cases-vertical** — вертикальные кейсы (поля под текущий макет карточек)
  - **cases-advertising** — рекламные кейсы (заголовок одной строкой, модалка с пунктами результата)
  - **secrets-posts** — статьи / «делимся секретами» (заголовок, slug, обложка, текст Lexical)

## Кейсы на главной

- Главная страница вызывает `getCasesForSite()` и передаёт данные в `<CasesSection />`.
- Если в коллекции **нет ни одной валидной записи** (или ошибка БД), подставляется **статический fallback** из `cases.data.ts`.
- Как только в админке есть хотя бы одна карточка в **Кейсы (вертикальные)** или **Кейсы (рекламные)** — на сайте показываются **только записи из CMS** для этой карусели (не смешиваются со статикой).
- Порядок карточек: поле **sortOrder** (меньше — раньше).
- Картинки: загрузите в **Media**, затем выберите в карточке кейса.
- ISR: в `page.tsx` задано `revalidate = 60` — главная пересобирает данные кейсов не чаще чем раз в 60 секунд (можно изменить).

## Локальный запуск

1. **`postinstall`** и **`predev`** запускают `scripts/ensure-env-local.mjs`: при необходимости создаётся или дополняется **`.env.local`** (валидный **`PAYLOAD_SECRET`** ≥32 символов, при отсутствии — **`DATABASE_URL=file:./payload.db`**). При желании можно вручную скопировать из **`.env.example`** и поправить значения.
2. `pnpm dev:fast` или `pnpm dev`
3. Откройте `http://localhost:3000/admin` и зарегистрируйте первого пользователя.

### Автосид кейсов (только development)

При первом запуске **`next dev`** в **`src/instrumentation.ts`** вызывается **`seedCasesFromStaticIfMissing()`**: из **`cases.data.ts`** в Payload добавляются вертикальные и рекламные кейсы (и медиа), если записей с теми же **`internalKey`** ещё нет. Отключить: **`PAYLOAD_AUTO_SEED=0`** в `.env.local`.

Если в логе **`[instrumentation] seed cases:`** видно **`index … already exists`** (SQLite) — локальная **`payload.db`** могла остаться в несогласованном состоянии после смены версии Payload. Для чистого старта на машине без ценных данных в CMS можно удалить **`payload.db`** (и журнал рядом, если есть) и снова запустить **`pnpm dev`**.

## Типы и CLI

- `pnpm run generate:types` — генерирует `src/payload-types.ts`.  
  На некоторых версиях Node (например 24+) команда Payload CLI может падать с `ERR_REQUIRE_ASYNC_MODULE`; в таком случае используйте **Node 22 LTS** для этой команды или обновите Payload, когда выйдет фикс.

## Прод на своём сервере

1. Соберите приложение: `pnpm build` → `pnpm start` (или Docker).
2. Задайте в окружении `PAYLOAD_SECRET` и `DATABASE_URL`.
3. Для продакшена рекомендуется **PostgreSQL** и адаптер `@payloadcms/db-postgres` вместо SQLite (отдельная задача миграции).
4. Загрузки медиа по умолчанию на диске; при нескольких инстансах — вынести в **S3-совместимое** хранилище (плагин Payload).

## Превью черновиков

Сейчас коллекции с `read: () => true` — контент сразу «публичный». Следующий шаг: поля `_status` / draft, **Next.js Draft Mode** и секретный query для превью — по мере подключения фронта к API.

## Подключение фронта

Главная получает кейсы через **`getCasesForSite()`** (Payload + fallback на **`cases.data.ts`**). Остальной контент по мере подключения — через REST (`/api/...`) или **`getPayload`** в Server Components.
